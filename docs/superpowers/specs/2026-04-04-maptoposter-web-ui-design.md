# MapToPoster Web UI & Docker 镜像设计文档

## 1. 背景与目标
本项目旨在为开源项目 [MapToPoster](https://github.com/originalankur/maptoposter) 提供一个开箱即用的 Web 管理面板，并将其打包成轻量级的 Docker 镜像，方便用户通过浏览器直接生成个性化地图海报。

### 1.1 核心目标
- **开箱即用：** 用户只需运行 `docker run` 即可访问 Web UI。
- **功能完备：** 支持城市搜索、实时预览、17 种预设主题选择。
- **历史管理：** 记录并管理已生成的海报，支持下载和查看。

## 2. 技术架构方案
选择 **方案 A (单体轻量级架构)**，最大限度地减少依赖并提高部署可靠性。

### 2.1 技术栈
- **Backend:** Python (FastAPI/Flask) — 直接集成 MapToPoster 核心逻辑。
- **Frontend:** Jinja2 (模板引擎) + Tailwind CSS (极简样式) + HTMX (无刷新交互)。
- **Map Engine:** OSMnx, Matplotlib (集成在后端)。
- **History Storage:** JSON/SQLite (记录元数据) + 本地文件系统 (保存 PNG/PDF)。

## 3. 界面设计 (Dashboard 布局)
采用“侧边栏 + 主预览区”的经典仪表盘布局。

### 3.1 布局组成
- **侧边栏 (Sidebar):** 
  - 城市搜索输入框。
  - 主题下拉选择器。
  - 高级参数 (DPI, 距离, 比例)。
  - “开始生成”按钮。
- **主区域 (Main Content):** 
  - 地图渲染预览 (显示当前进度或生成后的结果)。
  - 历史列表 (展示最近生成的几张海报缩略图)。

## 4. Docker 构建策略
采用 **方案 B (多阶段极简镜像)**。

### 4.1 构建过程
1. **Build Stage:** 安装编译依赖 (如 GCC, GEOS)，通过 `pip` 安装 `OSMnx` 及其庞大的依赖库。
2. **Final Stage:** 使用精简的 Python 运行时环境，仅复制已安装的库文件和 Web UI 代码。
3. **Optimizations:** 使用 `.dockerignore` 排除不必要的文件，层合并以减少体积。

## 5. 数据流与并发处理
由于地图生成是 **密集型 CPU 任务**：
- **异步处理：** 使用任务队列 (如简单的 Python 线程池) 处理生成任务，前端通过 HTMX 轮询显示实时进度。
- **持久化：** 通过 Docker Volume (映射 `/app/exports`) 挂载本地路径，确保容器重启后历史记录不丢失。

## 6. 成功标准
- [ ] 容器启动时间 < 30 秒 (不含首次地图下载)。
- [ ] 支持 300 DPI 海报生成并通过 Web 正常下载。
- [ ] 镜像体积控制在合理范围内 (目标 < 800MB)。

## 7. 风险评估
- **网络波动：** OSMnx 下载地图数据依赖外部 API，需增加超时重试和错误提示。
- **内存占用：** 大城市地图渲染可能消耗较多内存，需在文档中注明推荐的 Docker 内存配额。
