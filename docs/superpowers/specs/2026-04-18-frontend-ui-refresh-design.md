# MapToPoster Frontend UI Refresh Design

## Goal

将当前偏后台表单式的首页，重构为一个“画廊工作室”风格的海报生成器：首页既要保留第一次访问即可快速出图的效率，也要让预览、主题选择、历史作品和生成状态拥有更强的作品感与陈列感。

## Product Direction

- 定位：有作品气质的消费级生成器，而不是生产后台。
- 首要场景：第一次使用即可快速生成成品。
- 次要场景：生成后继续微调主题、尺寸和高级参数，反复试出满意版本。
- 语言风格：中文优先，少量英文只用于品牌或海报本身，不把界面做成英文导向产品。

## Design Decision Summary

- 页面骨架采用 `A` 方向：首屏为“大预览舞台 + 精简生成表单”的左右布局。
- 整体氛围采用浅色画廊感，页面像作品陈列页，海报本身是视觉主角。
- 高级参数采用“半展开”策略：首页保留最常用参数，其余内容收进精修面板。
- 最近生成记录从后台卡片列表改为更像作品带的展示区。
- 继续沿用现有 `Jinja2 + Tailwind + HTMX` 技术路线，不做前端框架迁移。

## Existing Problems

### 1. 首页结构更像后台而不是创作工具

当前页面由左侧长表单和右侧空预览区组成，信息层级比较平均。用户进入页面后首先看到的是大量字段和说明文本，而不是“可以得到一张什么样的作品”。

### 2. 预览区存在感不足

预览区只是一块白底虚线框，空状态与成功状态都缺少作品感，无法承担“首屏视觉主角”的角色。

### 3. 参数组织方式偏平铺

城市、主题、尺寸、导出格式、DPI、构图微调、文案覆盖等参数几乎都在同一层级出现。虽然功能完整，但第一次使用时的认知负担偏重。

### 4. 历史记录仍是后台式加载体验

历史区域更像数据列表，而不是一组最近作品。它没有强化“继续浏览、复用、对比上一张成品”的用户心智。

### 5. 状态设计不完整

空状态、生成中、失败态、成功态目前都比较薄弱。交互反馈更多依赖按钮文字变化，缺少围绕“作品正在生成”的中心化体验。

## Target Experience

用户访问首页后，第一眼先看到一个高质感的大海报舞台，再在左侧用最少的信息完成一次生成：

1. 输入地点与地区。
2. 选择主题与尺寸。
3. 点击生成。
4. 右侧舞台进入生成态并最终展示成品。
5. 用户可继续打开精修面板微调参数，或在最近作品区查看刚刚生成的海报。

这个流程应该像“轻量创作工作室”，而不是“填写一张参数表再等待结果”。

## Information Architecture

### Page Structure

首页重构后的模块顺序如下：

1. 顶部导航
2. 首屏 Hero Workspace
3. 最近作品带
4. 轻量帮助或使用提示

其中，首屏 Hero Workspace 是唯一必须进入首屏视野的核心区域，其余内容都为补充模块。

### Top Navigation

顶部导航保持轻量，只承担：

- 品牌名 `MapToPoster`
- 一句短说明，表达“将城市转化为可打印海报”
- 右侧一个轻量入口，例如“最近作品”锚点或简要帮助

导航不承载复杂菜单，避免破坏首页的聚焦感。

### Hero Workspace

首屏采用双栏布局：

- 左栏：快速生成区
- 右栏：海报舞台

桌面端布局必须明显偏向右侧，让预览成为视觉重心。移动端则回退为单列，顺序为：

1. 品牌与简述
2. 快速生成区
3. 海报舞台
4. 最近作品带

## Module Design

### 1. Quick Generate Panel

左栏不是传统后台侧边栏，而是一块轻量创作面板。它默认只展示：

- 地点名称
- 国家 / 省份 / 地区
- 主题选择
- 尺寸选择
- 生成按钮

此外保留 2 到 3 个对结果感知最强的轻量参数常驻。推荐保留：

- 主题
- 尺寸
- 海报文案语言

其余内容全部移入“精修设置”。

该模块要具备以下特性：

- 标题与说明更短，不再用大段提示文案压住视觉节奏。
- 输入区的标签在上方，帮助文案在下方。
- 城市搜索相关的长篇提示改写为单句高价值提醒，例如“中文搜索建议补全到市、区、县级，提高定位准确率。”
- 生成按钮应足够强，成为左栏唯一明显强调元素。

### 2. Poster Stage

右栏是整个首页的主角，承担以下状态：

- 空状态
- 生成中状态
- 成功预览状态
- 失败状态

其视觉目标是“作品舞台”，而不是“返回结果的容器”。

#### Empty State

空状态展示：

- 一个高质感海报占位框
- 一句简洁引导，如“输入地点，生成你的第一张城市海报”
- 一行补充说明，说明支持打印与导出

避免使用虚线框和弱存在感提示文案。

#### Generating State

生成中状态需要将反馈聚焦在舞台区域，而不是只改变按钮文字。右侧应显示：

- 海报比例骨架屏
- 当前生成步骤或进度文案
- 当前地点与主题摘要

如果后台只能提供有限日志，则将文案处理为“正在定位区域 / 正在下载地图数据 / 正在渲染海报”这类可理解的进度语义。

#### Success State

成功后，舞台直接展示成品大图，并在不打断画面完整性的前提下提供：

- 下载
- 再次生成
- 打开精修设置

下载与再次生成必须是最自然的后续动作。

#### Error State

错误反馈需要贴近舞台和输入区，不应只显示孤立红字。错误提示应分成：

- 主错误说明
- 可执行建议

例如：

- “没有找到这个地点”
- “建议补全到市、区、县级，或补充国家 / 省份信息后重试”

### 3. Theme Selector

主题选择不再只用普通下拉框承载。目标是让用户在不增加操作成本的前提下，更快理解风格差异。

推荐改为：

- 可点击的轻量主题条
- 或者紧凑的主题样式卡

每个主题项需要至少包含：

- 中文名称
- 颜色小样
- 选中态

主题选择仍然应保持高效率，不能演化成需要滚动浏览的大型画廊。

### 4. Refine Panel

高级参数统一收纳为一个可展开的“精修设置”模块。其内部按语义分组：

#### 输出相关

- 尺寸
- DPI
- 导出格式

#### 构图相关

- 地图范围
- 水平偏移
- 垂直偏移
- 道路粗细缩放

#### 文案相关

- 海报大标题
- 海报小标题
- 文案语言

该模块默认收起，但首屏可以露出少量已选摘要，例如：

- “已选：16x20 / 300 DPI / PNG”

让用户知道还有更多控制能力，而不会被完整长表单压住。

### 5. Recent Works Strip

最近生成记录改为更像“作品带”而不是后台卡片网格：

- 成品缩略图优先
- 城市与主题信息次之
- 时间信息最弱

交互目标：

- 浏览最近做过的作品
- 迅速点击查看或再次生成
- 强化“我刚刚完成了一组创作”的连续感

在桌面端推荐使用横向作品带或不等宽作品网格；在移动端可退化为纵向作品列表。

## Content Strategy

### Writing Style

- 中文优先
- 句子更短
- 减少工程化说明语气
- 把冗长提示改成“少量但重要”的帮助信息

### Examples

建议替换当前偏说明书式表达：

- “搜索目标区域（中英文皆可）”
  改为
  “地点”

- “海报文字覆盖（选填）”
  改为
  “海报标题”

- “智能缓存机制”
  改为
  更克制的提示文案，放在精修或帮助区，而不是占用主输入流

## Visual Language

### Overall Mood

整体采用浅色画廊感，页面背景类似温和展墙或纸张底色，而不是默认灰白后台。视觉焦点依次为：

1. 海报预览
2. 生成按钮
3. 主题与尺寸
4. 其余辅助信息

### Color System

- 基底颜色：温暖浅中性色
- 主要文字：深灰，不使用纯黑
- 强调色：只保留一种主强调色

推荐强调色方向：

- 克制陶土色
- 或低饱和松针绿

禁止使用常见 AI 产品式蓝紫荧光高亮。

### Typography

- 中文界面以稳定、现代的无衬线为主
- 标题与正文拉开层级，但不追求过度张扬
- 标题用于建立作品感，正文用于确保可读性

如果当前项目中没有合适的中文 Web 字体，则优先采用高质量系统字体栈，不为了追求“设计感”贸然增加网络依赖。

### Shape and Surface

- 大模块采用较柔和圆角
- 边框细而克制
- 卡片阴影要轻，避免后台式硬阴影堆叠
- 预览舞台可以拥有更强的层次感，但不应成为深色大面板

## Interaction Design

### Motion Principles

动效只服务于以下目的：

- 强化层级进入顺序
- 反馈用户操作
- 表达海报正在生成
- 平滑切换预览与作品列表

不做夸张炫技效果，不引入和品牌无关的“秀肌肉式”动画。

### Required Motion

- 首屏加载时，左侧面板、右侧舞台、最近作品依次淡入
- 主题切换有轻微选中反馈
- 精修面板展开 / 收起有平滑高度或透明度过渡
- 成功生成后，骨架屏平滑切换为真实海报预览
- 最近作品卡 hover 或点击时有轻量抬升 / 边框反馈

### Button Feedback

主按钮必须有明确 hover、active、busy 三种状态：

- hover：轻微颜色和阴影变化
- active：轻微压下反馈
- busy：按钮仍保持明确可见，但不出现突兀跳动

## State Design

### Empty State

- 预览区显示高质感占位
- 最近作品区若为空，显示“你的作品会出现在这里”式引导

### Loading State

- 预览区使用海报比例骨架
- 最近作品区使用作品卡骨架
- 历史加载文案不再单独裸露

### Busy State

- 生成按钮进入忙碌态
- 输入区保留已填写信息，便于用户确认当前任务
- 预览舞台承担主要反馈

### Success State

- 成品立即成为主视觉
- 下载入口清晰
- 支持继续微调重新生成

### Error State

- 错误信息靠近相关模块
- 给出明确可执行建议

## Responsive Strategy

### Desktop

- 使用双栏布局
- 预览区宽于表单区
- 最近作品区为横向作品带或多列作品网格

### Tablet

- 双栏可变为偏紧凑布局
- 精修面板内容按分组换行

### Mobile

- 首屏改为单列
- 快速生成区放在海报舞台前面
- 最近作品改为纵向列表
- 所有输入与按钮满足触屏舒适尺寸

## Technical Plan Boundary

本次设计阶段明确不做以下事项：

- 不迁移到 React 或 Next.js
- 不重写后端海报生成逻辑
- 不新增复杂前端状态管理
- 不将主题系统重构为独立配置平台

本次实现主要聚焦：

- `app/templates/base.html`
- `app/templates/index.html`
- `app/static/css/style.css`

必要时增加少量原生 JavaScript，用于：

- 面板展开收起
- 主题切换反馈
- 状态切换增强

但仍以 HTMX 与现有服务端渲染模式为主。

## Testing and Verification Expectations

实现阶段必须验证以下体验：

- 首页在桌面和移动端都能正常加载
- 第一次进入时，首屏视觉重心明确在海报舞台
- 生成按钮触发后，用户能清楚感知当前任务状态
- 成功生成后，下载与二次编辑路径明显
- 历史记录为空和非空两种状态都成立
- 地理编码失败时，错误信息可理解且靠近操作区域

## Open Questions Resolved In This Spec

以下方向已在本轮确认，后续实现不再重复讨论：

- 采用 `A` 骨架
- 采用浅色画廊感
- 首页服务首次快速出图
- 高级参数采用半展开
- 中文优先

## Implementation Handoff Notes

实现时要特别注意：

- 不要把页面重新做回后台风格
- 不要让主题选择变成低效的大画廊
- 不要让帮助文案重新膨胀成大段说明书
- 不要让右侧预览退化回一个普通容器

最终判断标准不是“功能都还在”，而是“用户是否明显感到这变成了一个更高级、更顺手的海报工作室”。
