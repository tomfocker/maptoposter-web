Docker 构建策略推荐：方案 B (极简生产镜像)

为什么推荐方案 B？

方案 B (专家推荐)

  • 体积更小： 只保留运行代码必需的文件，下载更快（GIS 库通常很大）。
  • 安全性高： 镜像内不含编译器等开发工具，减少攻击面。
  • 部署友好： 虽然构建脚本稍显复杂，但对最终用户来说，docker run 的体验是一样的。

方案 A

  • 镜像中会充斥着大量的构建缓存和临时文件。
  • 体积可能达到 2GB 以上，浪费存储和带宽。

最终方案预告

  • 界面： 方案 1 (Dashboard) - 左侧参数调整，右侧大图预览。
  • 架构： Flask/FastAPI + Jinja2 (轻量级)。
  • 数据持久化： 使用 Docker Volume 挂载海报导出目录，保持生成历史。
