3. 界面布局 & Docker 策略

我们已经选定了 Flask/FastAPI 架构。下面是具体的界面和镜像实现路径。

方案 1: 经典仪表盘 (Dashboard)
侧边栏: 创建、历史、设置
参数表单
海报实时预览区域

优点： 适合多图生成、对比，更有“应用感”。

方案 2: 引导式流程 (Flow)

1. 搜城市 -> 2. 选主题 -> 3. 生成预览 -> 4. 下载历史

优点： 及其直观，新用户几乎不需要学习成本。

Docker 构建策略

A

全能单体镜像 (All-in-one)

将 Python, OSMnx, Web Server 全部打入一个镜像。用户只需 docker run 即可在 8080 端口访问。

B

极简生产镜像 (Slim)

使用多阶段构建（Multi-stage），仅保留运行时必需的依赖，尽量缩减体积（考虑到 OSMnx 依赖较重，这很重要）。
